Creating Changes and Additions

Now that you are all set up, you can learn how to do work on the site.

Committing Changes

If you have done some work on the website and you want to push your changes to the repository, perform the following commands.

  1. Re-initialize a Git repository in your project folder by running a git initcommand in your terminal.

    1. When you run this command, you should get a “Re-initializing git repository in this location” message. If you do not get this message, or get a message that says “Initializing git repository in this location,” stop, as something went wrong in the cloning process.

  2. Add untracked files.

    1. If you have created new files or made changes to existing ones, you will need to add your changes.

      1. First, run the command git status to make sure that you are on the master branch, which reflects the production-ready state. This will also allow you to see what files have not yet been added, as they will appear in red. Added files will appear in green.

        2. The untracked files are displayed in red.

      2. To add files, run the git add command. For example, if you created or edited a file named “readMe1,” you would add the file by running a git add readME1.mdcommand.

        Add each file (readMe1, readMe2 and readMe3).

        Once added, they will appear green when you run the git statuscommand.

        Now that the files have been added, they will appear in green when you check their status.

    2. When you are ready to commit your changes, you can run the commit command, which includes a mandatory message detailing what action you have taken. Do this by running git commit -m “”in your terminal. (Inside of the quotation marks, you will type your message details.)

      For example, once you are ready to commit readMe1.md, you would run git commit -m “now I have pushed the files where we have resolved conflicts”in your terminal, or whatever message you choose.

      The files have been committed and include a message detailing the action taken.

  3. Push up your changes.

    1. To make the final step in pushing your changes to Git, where they will show up in your commit history and in the project directory on GitHub, run thegit push origin mastercommand in your terminal.

      The files have been pushed to master.

    2. If successful, when future collaborators pull the repository, your changes will have been included.
