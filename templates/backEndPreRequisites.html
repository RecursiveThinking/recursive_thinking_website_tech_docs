PreRequisites

The Following are the Prerequisites for the Recursive Thinking Backend:

  • Python

    Our deployment script (deployrtw.py), which runs the aws cli and processes the template.yml file into different Amazon Web Services, runs on Python, so you'll need to install the language if you haven't already.

    A quick way to test if you have python installed is to run:

    python --version

    in your CLI/terminal. If you don't see "Python x.x.x" (where the x's are version/release numbers) then you don't have Python installed. If this is the case, click on the link above to go to the Python Software Foundation's website and download a copy for your OS version.

    *Note - some of the code in deployrtw.py requires modules only available in Python 3. So please download a Python 3 version, or most likely, there will be errors in running the script. Some of us are running Python 3.6.5, and there seems to be no problems with this version.

  • Install Amazon Web Service Command Line Interface (AWS CLI)

    The deployment script executes AWS CLI commands in order to build the Backend Stack of Component Services.

    Please follow all the instructions in the link and make sure AWS CLI is properly installed for your operating system.

    Some useful commands, after you go through the documentation, to verify your installation is correct:

    aws --version

    this should return something like: (although yours maybe slightly differnet based on your versioning)
    aws-cli/1.11.84 Python/3.6.2 Linux/4.4.0-59-generic botocore/1.5.47

  • Configure Amazon Web Service Command Line Interface

    Once you install AWS CLI - you'll need to configure it so that it points to your Amazon Web Service Account. Please follow the instructions at the link above to do this.

    Please note that there has been some confusion with the instructions above when it gets to "To get the access key ID and secret access key for an IAM user" --> Item 3 ("Choose your IAM user name"). Creating an IAM Resource and Setting Permissions (needs anchor tag?)

    It is recommended that you check The AWS credentials and config File as described in the documentation.

    The credentials file should contain the AWS Access Key ID and the AWS Secret Access Key

    The config file should contain the region name and default output format.

    If these files have both these keys and the correct values from your account, you should have a properly configured AWS CLI!

Creating an IAM Resource and Setting Permissions

  • Step 01:

    Open IAM Console

    Select Users From Left Menu

    Click the "Add user" Button

  • Step 02:

    Fill in Desired Name

    Select "Programatic Access" checkbox

    Click "Next: Permissions"

  • Step 03a:

    Select the "Attach Existing Policies directly" div

    In the policy table, click the "AdministratorAccess" checkbox

    Step 03b:

    Under "Set Permissions boundary" - click the radio button - "Creat user without a permissions boundary"

    Click "Next: Review"
